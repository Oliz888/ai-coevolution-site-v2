About This Explorer

Methodology, data model, and codebook for the Humanoid Robot Imaginaries co-mention network.

Why

This interactive explorer accompanies a research project investigating how science-fiction humanoids shape — and are shaped by — real-world technology, policy, and public discourse. The central research question is:

Which fictional humanoids become canonical reference points in STS and humanities discussions of technology and society?

By constructing a co-mention network from a curated academic corpus, we can visualize which fictional characters and works recur across scholarship, how they cluster thematically, and which ones have become anchor points — canonical reference figures cited across many independent publications.

How

Corpus construction

The literature corpus (Version 2.1) was built through a multi-stage process:

  1. Initial search (v1) — keyword and topic-based searches across Google Scholar, Scopus, and institutional databases for STS, humanities, and social-science scholarship that mobilizes science-fiction humanoids, robots, androids, cyborgs, or AI characters in discussions of technology, society, ethics, governance, innovation, and futures. Produced 64 papers.
  2. Web of Science expansion (v2.0) — three systematic WoS queries (369 records screened) with LLM-aided metadata screening and full-text PDF reading. 46 new papers added: 10 from query 1, 8 from query 2, 28 from query 3. Corpus reached 110 papers.
  3. Backward citation — reference-list mining from included papers, particularly to identify foundational STS frameworks (e.g., Borup et al. 2006 on expectations, Jasanoff & Kim 2015 on imaginaries, Callon 1986 on ANT).
  4. OpenAlex systematic search (v2.1) — the same three WoS queries were translated to the OpenAlex API as a corpus validation round. The pipeline:
    1. Query execution — three boolean queries run against OpenAlex full-text search, returning 1,461 open-access articles and reviews after type filtering.
    2. Deduplication — removed papers already in the v2.0 corpus by DOI and title matching.
    3. Three-stage LLM screening — (a) programmatic pre-filter on title keywords, (b) strict science-fiction title relevance check, (c) AI/robots/humanoid requirement plus false-positive removal. Selection rate: 12%, matching the WoS screening rate. 168 candidates passed.
    4. Priority ranking — candidates ranked by citation count, character/work density, and topical relevance into HIGH (48) and MEDIUM (120) priority tiers.
    5. PDF acquisition — open-access PDFs downloaded automatically where available; remaining papers downloaded manually via institutional access.
    6. Full-text extraction — each PDF scanned for character and work mentions using 60+ character patterns and 70+ work patterns with context-aware matching to avoid false positives.
    137 new papers added, bringing the corpus to 247 papers.
  5. Forward citation search — the 64 v1 papers were traced forward via OpenAlex to find 394 SF-relevant citing papers, cross-checked against the corpus for additional candidates.
  6. Screening criteria — papers must engage with fictional humanoid/robot/AI characters in a technology-and-society framing; pure literary criticism, pure engineering, and non-English works (unless highly cited) are excluded.
  7. Character and work extraction — each paper is read to identify all fictional characters and works mentioned, along with their narrative valence and thematic motifs.

Each paper's discovery method is tracked in a discovery_source field and corpus version in corpus_version for methodological transparency.

Network construction

Two networks are built from the extracted data:

  • Character co-mention network — two characters are connected if they appear in the same paper. Edge weight = number of shared papers.
  • Work co-mention network — two works are connected if they appear in the same paper. Edge weight = number of shared papers.

The networks are rendered using vis.js with a Barnes-Hut force-directed layout. Node size is proportional to mention count (number of papers). Node color reflects valence.

Nodes and Edges

What is a node?

Each node represents either a fictional character (e.g., HAL 9000, The Terminator, Astro Boy) or a fictional work (e.g., Blade Runner, 2001: A Space Odyssey), depending on which view is active.

PropertyEncodingMeaning
Size Proportional to count Number of papers in the corpus that mention this character/work. Larger nodes appear in more publications.
Color Green / Red / Orange / Gray Overall valence of the character: utopian (hopeful, emancipatory), dystopian (threatening, controlling), mixed (both), or neutral (not clearly coded).

What is an edge?

An edge connects two nodes that co-occur in the same paper. If Paper X mentions both HAL 9000 and The Terminator, an edge links them. Edge weight (thickness) equals the number of papers in which both entities appear together.

High-weight edges reveal which characters or works are frequently discussed together in scholarship — indicating thematic clusters, comparative analysis patterns, or canonical groupings.

Metadata Sidebar Codebook

When you click a node in the network, the right panel displays detailed metadata. Here is what each section contains:

1. Valence badge

BadgeMeaning
utopianCharacter is predominantly portrayed as beneficial, helpful, emancipatory, or hopeful in the literature.
dystopianCharacter is predominantly portrayed as threatening, controlling, dehumanizing, or destructive.
mixedCharacter is discussed with both utopian and dystopian framings across papers.
neutralCharacter's portrayal is not clearly utopian or dystopian, or is insufficiently characterized in the corpus.

2. Wikipedia / Wikidata

Automatically enriched metadata from Wikipedia and Wikidata for each character, including:

  • Thumbnail — representative image from Wikipedia
  • Description — Wikidata short description
  • Extract — opening paragraph of the Wikipedia article
  • Structured fields — type (film, character, novel), creator, director, series, genre, publication date, country of origin, awards, and other Wikidata properties

6. Papers

All papers in the corpus that mention this character/work, sorted by priority then year. Each card shows:

  • Citation key — e.g., OsawaEtAl2022

  • Flags — "desire" (character linked to desire/aspiration attributes) or "invention" (character linked to real-world invention/R&D)
